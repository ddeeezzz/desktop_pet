### Windows 桌宠项目功能规划

---

#### 1. 鼠标穿透与交互模式调整

- **目标：** 使用多边形实现背景穿透，并调整交互逻辑。
- **具体功能点：**
    - **窗口设置：**
        - 设置窗口为无边框，始终保持在最顶层，并启用透明背景。
        - **核心改变**：使用一个 `CollisionPolygon2D` 节点来定义可点击区域（宠物本身），启用窗口鼠标穿透 (`mouse_passthrough = true`)，其他透明区域则自动穿透。
    - **交互逻辑变更：**
        - **单击交互：** 单击桌宠的多边形区域将触发特定动画或行为。
        - **右键：** 在多边形区域右键点击，唤出菜单。
        - **拖动交互：** 当用户在多边形区域内按下并拖动鼠标时，窗口跟随移动。
    - **菜单内容：**
        - **“拖动”选项：** 用户点击后，窗口进入可拖动模式。
        - **“设置”选项：** 打开独立的设置面板。
        - **“退出”选项：** 关闭应用。
    - **拖动功能：**
        - **状态切换：** 拖动时，不需要改变 `mouse_passthrough` 状态，因为拖动事件只在多边形区域内被捕获。
        - **拖动结束：** 用户松开鼠标左键，拖动状态结束。
    - **设置与形象更换：**
        - **菜单入口：** “设置”作为右键菜单的一个选项。
        - **界面功能：** 打开一个独立的窗口，其中包含所有输入框和按钮，用于更改精灵表、`hframes`、`vframes`、`frames`、`fps` 等参数，并同步更新配置文件。该窗口的大小不受主宠物窗口的限制。

#### 2. 设置界面功能

- **目标：** 创建一个功能完整的设置界面，允许用户修改宠物动画参数。
- **具体功能点：**
    - **UI 布局：**
        - 设置菜单中应包含标签和对应的数字输入框（`SpinBox` 或 `LineEdit`），分别用于调整 `hframes`、`vframes`、`frames` 和 `fps`。
    - **数据展示与输入：**
        - 打开设置界面时，输入框自动填充当前动画的参数值。
        - `hframes`、`vframes` 和 `frames` 的输入框只接受整数。
        - `fps` 的输入框接受一位小数的浮点数。
    - **配置应用：**
        - 提供一个“确定”或“应用”按钮。
        - 只有点击该按钮后，新的动画参数和精灵表路径才会生效。
        - 修改后的参数将同步保存到配置文件中，确保下次启动时加载最新设置。

#### 3. “云朵”UI 与菜单控制

- **目标：** 实现鼠标悬浮显示 UI 元素，并以此作为菜单的入口。
- **具体功能点：**
    - **悬浮计时：**
        - 使用 `Timer` 节点来检测鼠标悬停时长。
        - 鼠标进入桌宠区域时启动计时，离开时重置。
    - **云朵 UI 控制：**
        - 当计时器达到一秒时，使“云朵”UI 元素可见。
    - **菜单显示/隐藏：**
        - 设置菜单在初始状态下不可见。
        - 点击可见的“云朵”后，设置菜单变为可见。
        - 需实现一个关闭菜单的逻辑，例如在菜单上添加一个“关闭”按钮。
